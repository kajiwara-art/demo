Stage by Stage
工程別 AI活用リファレンス
各工程の「AIに任せられること」と、そのまま使えるプロンプト例。
01
初回接触・面談準備
CA業務｜登録者への最初のアプローチ
✉️
初回連絡文・SMS・メールの型作成
登録経路（サイト登録/スカウト返信）や職種（PT/OT/ST）別に、返信率の高い初回連絡文の型を作っておく。時間帯・トーン別のバリエーションも一括生成
🔍
面談前の仮説メモ生成
登録情報（経験年数・現職場の種別・希望エリア）から「この人が転職を考える背景の仮説」「聞くべき質問リスト」を面談前に3分で用意
📅
日程調整文・リマインド文の自動化
面談前日のリマインド、キャンセル時の再設定文など、定型コミュニケーションをすべて型化
PROMPT EXAMPLE
回復期病院で5年目のPT、28歳、訪問リハ希望で登録。この方が転職を考えている背景として考えられる仮説を5つ、面談で確認すべき質問を10個、優先順に出して。給与・キャリア・働き方・人間関係・スキルの観点を網羅して。
02
キャリア面談
CA業務｜ヒアリングと信頼構築
📝
面談メモ→構造化サマリーの自動変換
面談中の走り書きメモを「転職理由/希望条件(MUST・WANT)/懸念/温度感/次アクション」に自動整理。システム入力用の要約も同時生成
🧭
キャリアパスの選択肢提示資料
「回復期→訪問」「病院→通所」など、リハ職特有のキャリアチェンジで変わること（給与構造・働き方・スキル・算定単位）を比較表で作成し、面談中にそのまま見せられる形に
💬
難しい相談への回答準備
「ブランク3年あるが復帰できるか」「管理職経験なしで年収を上げたい」など、答えにくい相談パターンへの回答骨子を事前にストック化
PROMPT EXAMPLE
以下の面談メモを整理して。①転職理由（本音と建前を区別）②希望条件をMUST/WANTに分類 ③懸念・リスク ④温度感（5段階）⑤次回までのアクション。最後にシステム登録用の200字サマリーも。［メモ貼り付け］
03
マッチング・求人提案
両面の強みが最も活きる工程
🎯
求人票→求職者別「刺さる紹介文」変換
同じ求人でも、Aさんには「教育体制」、Bさんには「残業の少なさ」を軸に紹介文を書き分け。求職者プロフィールと求人票を渡せば個別最適化された提案文が完成
⚖️
複数求人の比較表を面談用に自動生成
候補3件の「給与/通勤/症例/教育/昇給イメージ/職場の雰囲気」比較表を数分で作成。求職者の意思決定を早める
🔁
ミスマッチの事前検知
両面担当だからこそ知っている事業所側の実情（教育体制の実態・退職者の傾向）と求職者の希望を突き合わせ、「この組み合わせの懸念点」をAIに洗い出させてから提案
PROMPT EXAMPLE
この求人票を、次の求職者向けの紹介メッセージに変換して。求職者：回復期5年目PT・訪問未経験・年収up希望・運転苦手ではない・教育体制を気にしている。不安を先回りでケアする一文を必ず入れて。300字以内。［求人票貼り付け］
04
書類添削・面接対策
CA業務｜選考通過率を上げる
📄
職務経歴書の添削・リライト
リハ職の経歴書にありがちな「業務の羅列」を、症例数・介入の工夫・多職種連携・実習指導などの成果ベースに書き換え。応募先の種別（回復期/訪問/通所）に合わせた強調点の調整も
🎤
面接想定問答の個別生成
応募先の種別×求職者の経歴から「聞かれそうな質問と回答骨子」を生成。転職回数が多い・ブランクありなどの懸念への切り返しも準備
📨
推薦文の品質向上
事業所に送る推薦文を、面談メモから自動起草。「なぜこの人がこの職場に合うか」を事業所の採用観点で書けるのは両面型の強み
PROMPT EXAMPLE
訪問リハ事業所への応募面接を想定して。候補者：回復期5年目PT、訪問未経験。「なぜ訪問か」「一人で判断できるか」「利用者家族との関係構築」を軸に想定質問8個と、経歴を踏まえた回答例を作って。未経験の不安を強みに変換する言い回しで。
05
事業所開拓・求人ヒアリング
RA業務｜求人の質が成約を決める
🔎
訪問前の事業所リサーチ自動化
法人HP・介護事業所情報公表システム・求人履歴から「事業所の特徴・採用に苦戦していそうな点・提案の切り口」を訪問前に整理。初回訪問の質が変わる
📋
求人ヒアリングシートの深掘り質問生成
「PT募集・年収400万」で終わらせず、教育体制・症例構成・残業実態・前任者の退職理由など、求職者が本当に知りたい情報を引き出す質問リストを準備
✍️
「選ばれる求人票」へのリライト
事業所からもらった情報を、求職者目線で魅力が伝わる求人票に変換。曖昧な「アットホームな職場」を具体的な事実（歓迎会の頻度・院内勉強会・有給消化率）に置き換える提案も
PROMPT EXAMPLE
明日訪問する訪問看護ステーションの初回商談準備をしたい。この事業所HP情報から、①事業所の強み・特徴 ②リハ職採用で苦戦していそうな仮説 ③ヒアリングすべき質問10個 ④他社エージェントと差別化できる提案の切り口を整理して。［HP情報貼り付け］
06
推薦・条件調整・クロージング
両面の情報を持つあなたが最も価値を出せる工程
🤝
条件交渉のシナリオ準備
「求職者は年収420万希望・事業所の提示は390万」のような場面で、双方の落とし所・代替条件（賞与/手当/昇給テーブル/研修費）・交渉の順番をAIと壁打ちしてから臨む
💭
迷っている求職者への後押し整理
「決めきれない」求職者の迷いポイントを言語化し、意思決定を助ける比較材料・不安への回答を準備。無理押しではなく「判断材料を揃える」型
📄
内定条件の確認・すり合わせ文書
口頭で決まった条件（給与・勤務地・オンコール有無・訪問件数ノルマ）を漏れなく文書化し、入職後トラブルを予防
PROMPT EXAMPLE
条件交渉の壁打ち相手になって。求職者：希望年収420万・現職380万・訪問未経験。事業所：提示390万・インセンティブ制度あり・人手不足で採用意欲高。双方が納得できる着地シナリオを3パターン、交渉の切り出し方の実際のセリフ付きで。
07
入職後フォロー・リピート獲得
CA業務｜定着支援と紹介の連鎖
🌱
フォロー連絡の仕組み化
入職1週間・1ヶ月・3ヶ月のタイミング別フォロー文を型化。回答から「早期離職リスクの兆候」をAIに判定させ、優先フォロー対象を絞る
🔗
紹介依頼(リファラル)の自然な打診文
満足度の高い入職者への「同僚・後輩の紹介」打診を、押し付けがましくないトーンで型化
🏥
事業所への定着レポート
入職者の定着状況を事業所に共有する文書を型化。次の求人依頼につながる信頼構築ツールとして
08
自己管理・KPI・業界学習
両面型の膨大なタスクを回す土台
📊
週次の数字振り返り
面談数・提案数・推薦数・内定数・成約数のファネルをAIに渡し、「どこで詰まっているか」「来週何を増やすべきか」の分析を毎週5分で
🗂️
タスク・案件の優先順位整理
両面型は同時進行案件が多い。毎朝、進行中案件リストから「今日絶対動かすべき3件」をAIに整理させる朝会習慣
📚
業界知識のキャッチアップ
診療報酬・介護報酬の改定、リハ職の算定単位、訪問リハと訪問看護Ⅰ5の違いなど、CAとしての信頼を左右する専門知識をAIに「新人CA向けに解説させる」学習法。面談で聞かれて答えられなかったことをその日のうちに潰す
PROMPT EXAMPLE
リハビリ職専門のキャリアアドバイザー新人だと思って教えて。「訪問リハビリ」と「訪問看護ステーションからのリハ職訪問」の違いを、制度・報酬・働き方・求職者へ説明するときの言い方の順で。面談でそのまま使える説明トークも付けて。
How to Build
AI従業員の作り方 — 「毎回頼む」から「一度作って使い回す」へ
上のプロンプトを毎回打つのは「壁打ち」。同じ指示を記憶させた専用AIを一度作れば、以降は素材を渡すだけになります。
ChatGPT → カスタムGPT / プロジェクト
画面左の「GPTを探す」→「作成する」。指示文と参考ファイルを登録すれば専用AIが完成。有料プランで利用可。手軽さNo.1で最初はこれがおすすめ
Gemini → Gem
「Gemマネージャー」から作成。仕組みはカスタムGPTと同じ。Googleカレンダー・Gmailと連携した日程調整系に強い
Copilot → エージェント
会社支給がMicrosoft環境ならこれ。社内データを外に出さずに使える可能性が高く、業務利用の本命。社内のAI利用ルールを最初に確認
Claude → プロジェクト / Claude Code サブエージェント
Claudeの「プロジェクト」はカスタムGPTと同じ感覚で作成可能。さらにClaude Codeなら複数のサブエージェントをチーム化でき、「一つの指示で複数AIが分担して動く」上級の使い方ができる（下記参照）
🚀 上級編：Claude Codeでのエージェント作成
Claude Codeでは、プロジェクトフォルダに設定ファイルを置くだけでAI従業員（サブエージェント）を定義できます。単体のカスタムGPTと違い、複数のエージェントを連携させて「面談メモ整理→求人マッチング案→推薦文下書き」を一つの指示で流すことも可能です。
①CLAUDE.md を作る——プロジェクトフォルダ直下に置くメモ。あなたの業務前提（リハ職CA・両面型・個人情報ルール）を書いておくと、毎回説明せずに全エージェントが前提を共有する
②.claude/agents/ にエージェント定義を置く——「面談メモ整理.md」「推薦文作成.md」のように1ファイル1エージェント。中身は上のテンプレと同じ（役割・手順・ルール）
③「/agents」コマンドで対話的に作成も可能——Claude Codeに「面談メモを整理するエージェントを作って」と頼めば、定義ファイル自体をClaudeが書いてくれる
④ファイルを直接読み書きできるのが最大の強み——面談メモのテキストファイルを置いたフォルダを渡せば、複数件をまとめて整理・比較表作成・週次振り返りまで一括処理できる
AGENT FILE EXAMPLE — .claude/agents/mendan-seiri.md
--- name: mendan-seiri description: 面談メモを構造化サマリーに整理する。面談メモの整理を頼まれたら必ずこのエージェントを使う。 --- あなたはリハビリ職専門・両面型CAのアシスタント。 渡された面談メモを以下の形式で整理する： 1. 転職理由（本音/建前） 2. 希望条件（MUST/WANT） 3. 懸念 4. 温度感（5段階） 5. 次アクション 6. 200字サマリー ルール：推測で補わない。個人名は「候補者」に置換。
💡 使い分けの目安：日常の1件ずつの処理はカスタムGPT/Gemで十分。複数件の一括処理・ファイル管理・エージェント連携をしたくなったらClaude Codeへ。
作成の5ステップ（1体あたり30分〜1時間）
1
任せる仕事を1つに絞る——「面談メモ整理だけ」「推薦文だけ」。欲張って何でも屋にすると精度が落ちる。1体1業務が原則
2
お手本を2〜3個集める——自分が「これは良い」と思う推薦文・面談サマリー・求人紹介文の実例（個人情報は伏せる）。お手本の質がAI従業員の質を決める
3
指示文（役割・手順・出力形式・禁止事項）を書く——下のテンプレをコピーして穴埋め。指示文自体をAIに下書きさせてもOK
4
過去の実案件3件でテスト——結果が自分の書いたものと比べてどうか採点。ズレた部分を指示文に追記（「〜はしない」「〜の順で書く」）
5
実務で使いながら育てる——違和感があったらその場で指示文を1行直す。2週間使えば「自分の分身」の精度になる
指示文テンプレート（例：面談メモ整理AI従業員）
INSTRUCTION TEMPLATE — そのままカスタムGPT/Gemの指示欄に貼って穴埋め
# 役割 あなたはリハビリ職（PT/OT/ST）専門人材紹介の両面型キャリアアドバイザーのアシスタントです。私の面談メモを構造化するのが仕事です。 # 手順 渡された面談メモを読み、以下の形式で整理する。 1. 転職理由（本音と建前を区別する） 2. 希望条件（MUST / WANT に分類） 3. 懸念・リスク（選考や定着に影響しそうな点） 4. 温度感（5段階＋根拠） 5. 次アクション（期限つき） 6. システム登録用200字サマリー # ルール - メモにない情報を推測で補わない。不明点は「要確認」として列挙する - 求職者を否定的に描写しない（事実ベースで書く） - 個人名が含まれていたら「候補者」に置換して出力する # 口調 簡潔なビジネス文体。箇条書き中心。
💡 このテンプレの「役割」「手順」「ルール」を差し替えれば、推薦文AI・求人票リライトAI・想定問答AIも同じ型で作れます。まずは工程02の面談メモ整理から作るのがおすすめ（毎日使う×効果が分かりやすい）。
Ground Rules
AI活用の絶対ルール
人材紹介は個人情報のかたまり。守りを固めてこそ攻められます。

運用上の注意

  • 求職者の氏名・連絡先・生年月日など個人を特定できる情報はAIに入力しない（「回復期5年目PT・28歳」のような属性化で十分機能する）
  • 事業所の非公開情報・条件は社内ルールを確認してから。会社が認めたAIツール（Copilot等の法人契約）が何かを最初に確認する
  • AIの出力する制度・報酬の情報は改定で古くなる。制度説明は必ず一次情報（厚労省資料）で裏取りしてから求職者に伝える
  • 推薦文・交渉文など「あなたの名前で出るもの」は必ず自分の目で最終確認。AIは下書き担当、判断と責任はあなた
