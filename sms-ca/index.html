01
初回接触・面談準備
CA業務｜登録者への最初のアプローチ
✉️
初回連絡文・SMS・メールの型作成
登録経路（サイト登録/スカウト返信）や職種（PT/OT/ST）別に、返信率の高い初回連絡文の型を作っておく。時間帯・トーン別のバリエーションも一括生成
🔍
面談前の仮説メモ生成
登録情報（経験年数・現職場の種別・希望エリア）から「この人が転職を考える背景の仮説」「聞くべき質問リスト」を面談前に3分で用意
📅
日程調整文・リマインド文の自動化
面談前日のリマインド、キャンセル時の再設定文など、定型コミュニケーションをすべて型化
PROMPT EXAMPLE
あなたはリハ職転職支援10年のベテランCA。以下の登録者の面談準備を手伝って。 【登録情報】回復期病院5年目のPT・28歳・訪問リハ希望・現年収380万 【依頼】 1. 転職背景の仮説を5つ。それぞれ「根拠（この属性の人によくあるパターン）」と「確からしさ（高/中/低）」つきで 2. 仮説を検証する質問を10個。ただし── ・最初の3つは答えやすい事実質問（アイスブレイク兼用） ・中盤は深掘り質問（「なぜ今なのか」を掘る） ・本人が言い出しにくいこと（人間関係・評価への不満）は直接聞かず、間接的に引き出す聞き方に変換 3. この人が面談で「言わなさそうだけど実は重要なこと」を2つ予想して 【形式】面談中にチラ見できるよう、A4半分に収まる箇条書きで
02
キャリア面談
CA業務｜ヒアリングと信頼構築
📝
面談メモ→構造化サマリーの自動変換
面談中の走り書きメモを「転職理由/希望条件(MUST・WANT)/懸念/温度感/次アクション」に自動整理。システム入力用の要約も同時生成
🧭
キャリアパスの選択肢提示資料
「回復期→訪問」「病院→通所」など、リハ職特有のキャリアチェンジで変わること（給与構造・働き方・スキル・算定単位）を比較表で作成し、面談中にそのまま見せられる形に
💬
難しい相談への回答準備
「ブランク3年あるが復帰できるか」「管理職経験なしで年収を上げたい」など、答えにくい相談パターンへの回答骨子を事前にストック化
PROMPT EXAMPLE
面談メモを以下の形式で整理して。 【整理形式】 ① 転職理由 — 「本人の言葉」と「言葉の裏にありそうな本音」を分けて。本音は推測と明記 ② 希望条件 — MUST（これが無いと決めない）/ WANT（あれば嬉しい）/ 譲歩可能 の3分類 ③ 懸念・リスク — 選考で不利になりうる点＋定着リスク。それぞれ「先回りの打ち手」つきで ④ 温度感 — 5段階。基準：5=今月中に決めたい / 4=良い求人があれば即動く / 3=比較検討中 / 2=情報収集 / 1=様子見。判定根拠の発言を引用すること ⑤ 次アクション — 「私がやること」「候補者にやってもらうこと」を分けて、期限つきで ⑥ システム登録用200字サマリー — 誰が読んでも状況がわかる文章で 【ルール】メモにない情報は推測で補わない（推測する場合は「推測：」と明記）。個人名は「候補者」に置換。最後に「このメモで確認し忘れていること」を3つ指摘して。 ［メモ貼り付け］
03
マッチング・求人提案
両面の強みが最も活きる工程
🎯
求人票→求職者別「刺さる紹介文」変換
同じ求人でも、Aさんには「教育体制」、Bさんには「残業の少なさ」を軸に紹介文を書き分け。求職者プロフィールと求人票を渡せば個別最適化された提案文が完成
⚖️
複数求人の比較表を面談用に自動生成
候補3件の「給与/通勤/症例/教育/昇給イメージ/職場の雰囲気」比較表を数分で作成。求職者の意思決定を早める
🔁
ミスマッチの事前検知
両面担当だからこそ知っている事業所側の実情（教育体制の実態・退職者の傾向）と求職者の希望を突き合わせ、「この組み合わせの懸念点」をAIに洗い出させてから提案
PROMPT EXAMPLE
この求人票を、以下の求職者専用の紹介メッセージに変換して。 【求職者】回復期5年目PT・訪問未経験・年収up希望・教育体制を気にしている・温度感4（良い求人なら即動く） 【メッセージ設計】 ・書き出し：この人の希望と求人の接点から入る（「求人のご案内です」で始めない） ・中盤：求人の魅力を「この人にとっての意味」に翻訳する（例：「教育体制充実」→「訪問未経験でも最初の3ヶ月は同行がある」） ・不安の先回り：訪問未経験の懸念に触れて、それが問題にならない根拠を1つ ・結び：返信のハードルを下げる一文（「話だけでも」レベル） 【禁止】「アットホーム」「働きやすい」等の中身のない形容詞／絵文字の多用／煽り 【出力】300字版と、SMSで送れる100字版の2パターン。件名案も3つ。 最後に自己チェック：「この文章はこの人以外にも送れてしまう汎用文になっていないか？」を判定して、なっていたら書き直し。 ［求人票貼り付け］
04
書類添削・面接対策
CA業務｜選考通過率を上げる
📄
職務経歴書の添削・リライト
リハ職の経歴書にありがちな「業務の羅列」を、症例数・介入の工夫・多職種連携・実習指導などの成果ベースに書き換え。応募先の種別（回復期/訪問/通所）に合わせた強調点の調整も
🎤
面接想定問答の個別生成
応募先の種別×求職者の経歴から「聞かれそうな質問と回答骨子」を生成。転職回数が多い・ブランクありなどの懸念への切り返しも準備
📨
推薦文の品質向上
事業所に送る推薦文を、面談メモから自動起草。「なぜこの人がこの職場に合うか」を事業所の採用観点で書けるのは両面型の強み
PROMPT EXAMPLE
あなたは訪問リハ事業所の採用面接官。以下の候補者の面接対策を作って。 【候補者】回復期5年目PT・訪問未経験・28歳 【作るもの】 1. 想定質問8個 — 「なぜ訪問か」「一人で判断できるか」「家族との関係構築」を軸に、確率順で。各質問に「面接官がこの質問で本当に見ているもの」を添える 2. 各質問の回答骨子 — STAR形式（状況→課題→行動→結果）。回復期の経験を訪問の文脈に翻訳する言い回しで。丸暗記用の文章ではなく「話す順番と入れるべき要素」のメモ形式 3. 逆質問5個 — 「調べればわかること」はNG。意欲と見極めの両方が伝わるもの 4. 危険な回答パターン3つ — この候補者がやりがちな失敗（例：回復期の自慢に聞こえる話し方）と回避法 【仕上げ】最後に面接官として一番厳しいツッコミを1つして、その切り返しも用意して。
05
事業所開拓・求人ヒアリング
RA業務｜求人の質が成約を決める
🔎
訪問前の事業所リサーチ自動化
法人HP・介護事業所情報公表システム・求人履歴から「事業所の特徴・採用に苦戦していそうな点・提案の切り口」を訪問前に整理。初回訪問の質が変わる
📋
求人ヒアリングシートの深掘り質問生成
「PT募集・年収400万」で終わらせず、教育体制・症例構成・残業実態・前任者の退職理由など、求職者が本当に知りたい情報を引き出す質問リストを準備
✍️
「選ばれる求人票」へのリライト
事業所からもらった情報を、求職者目線で魅力が伝わる求人票に変換。曖昧な「アットホームな職場」を具体的な事実（歓迎会の頻度・院内勉強会・有給消化率）に置き換える提案も
PROMPT EXAMPLE
明日訪問する訪問看護ステーションの初回商談準備。あなたは医療介護業界の法人営業コンサルタントとして分析して。 【分析してほしいこと】 ① 事業所プロファイル — 規模感・サービス特徴・運営母体を3行で ② 採用苦戦仮説 — 「リハ職採用でこの事業所が困っていそうなこと」を3つ。各仮説に根拠（HP・求人情報のどこからそう読めるか）と、当たっていた場合の提案の方向性をセットで ③ ヒアリング質問10個 — 順番も設計して。前半：相手が話しやすい現状質問 → 中盤：課題を引き出す質問 → 終盤：予算・決裁・時期のBANT確認。質問ごとに「何を判断するための質問か」を添える ④ 差別化トーク — 他社エージェントがやりがちな営業と、両面型CAだから言える一言の対比で2パターン ⑤ 商談の第一声 — HPの内容に触れた「調べてきた感」のある冒頭挨拶を1つ 【最後に】この事業所に紹介するとしたら、どんな人物像のリハ職が定着しそうかのペルソナも作って。 ［HP・求人情報貼り付け］
06
推薦・条件調整・クロージング
両面の情報を持つあなたが最も価値を出せる工程
🤝
条件交渉のシナリオ準備
「求職者は年収420万希望・事業所の提示は390万」のような場面で、双方の落とし所・代替条件（賞与/手当/昇給テーブル/研修費）・交渉の順番をAIと壁打ちしてから臨む
💭
迷っている求職者への後押し整理
「決めきれない」求職者の迷いポイントを言語化し、意思決定を助ける比較材料・不安への回答を準備。無理押しではなく「判断材料を揃える」型
📄
内定条件の確認・すり合わせ文書
口頭で決まった条件（給与・勤務地・オンコール有無・訪問件数ノルマ）を漏れなく文書化し、入職後トラブルを予防
PROMPT EXAMPLE
条件交渉の壁打ち相手になって。あなたは交渉学の専門家で、人材紹介の実務にも詳しい。 【状況】 ・求職者：希望年収420万／現職380万／訪問未経験／この求人への志望度は高い ・事業所：提示390万／インセンティブ制度あり／人手不足で採用意欲高／過去に年収で辞退された経験あり 【分析→シナリオの順で】 1. まず双方の本当の関心を分析：求職者は「420万」が目的か、それとも「現職より明確に上がった実感」か？事業所の390万は上限か初回提示か？ 2. 交渉の材料の棚卸し：基本給以外に動かせる変数を列挙（インセンティブ保証・昇給時期の前倒し・研修費・訪問件数の段階設定など） 3. 着地シナリオ3パターン — 松（求職者最優先）/竹（現実解）/梅（決裂回避の最低ライン）。各シナリオに「事業所への切り出しセリフ」「求職者への伝え方セリフ」の両方 4. 危険察知：この交渉が壊れるとしたらどこか。地雷ワードと、言ってはいけないタイミング 【ロールプレイ】最後に、あなたが渋い事業所役になって私と交渉の練習をして。私の言い方の改善点をその都度指摘して。

業界知識のキャッチアップ

PROMPT EXAMPLE
私はリハビリ職専門CAの新人。「訪問リハビリ」と「訪問看護ステーションからのリハ職訪問」の違いを教えて。 【教え方の指定】 1. まず30秒で言える要約（面談中にサッと説明できるレベル） 2. 次に制度・報酬・働き方の違いを表で 3. 求職者に説明するときのトーク例 — 「PTさんに説明する版」と「制度に詳しくない家族に説明する版」の2種類 4. 新人CAがやりがちな誤解・間違った説明を3つ 5. 最後に理解度チェックのクイズを3問出して。私が答えたら採点して 【ルール】制度・報酬の数字には「※改定で変わるため要一次情報確認」を必ず添えて。根拠となる制度名（介護保険/医療保険のどちらの枠か等）も明記して。

A
競合マップと差別化ポイント
「他社も同じ求人を持っている」前提で、何で選ばれるか
競合 強み 構造的な弱み（＝あなたの攻めどころ）
PTOTSTワーカー
トライト系		 求人数6万件超で業界最大級。全国カバー 量で勝負するため1人あたり対応が薄くなりがち。「数を送られて疲れた」層を丁寧さで取る
マイナビ
コメディカル		 ブランド力。アドバイザーが施設訪問し内部情報を把握 分業型が基本。求職者担当と法人担当の間で情報が減衰する。両面型の一気通貫で差をつける
レバウェル
リハビリ		 LINE中心の軽いコミュニケーションで若年層に強い キャリア相談の深さでは勝てる。30代以降・キャリアの岐路にいる層で優位に立つ
ジョブメドレー等
掲載型サイト		 事業所側の費用が安い（成功報酬が低額 or 掲載課金） 事業所は「応募は来るが見極めと調整は自前」。スクリーニング・条件調整・定着支援の代行価値で対抗
ハローワーク・
直接応募		 事業所の費用ゼロ。地元採用に強い。熱意が伝わる 最大の競合はここ。「紹介手数料に見合う価値」を言語化できないと事業所開拓で負ける（下の失注対策へ）
あなたの差別化の核（3つ）：①両面型＝面談で聞いた本音と事業所の実情を一人の頭の中で突き合わせられる（分業型は構造的に不可能） ②AI活用で対応の速さ×個別具体性を両立（他CAが1日かける準備を1時間で） ③リハ職のキャリア文脈（回復期→訪問等の制度・報酬まで）を語れる専門性
PROMPT — 週次の競合ウォッチ
私はリハ職専門の両面型CA。今週、候補者から「〇〇（競合名）にも登録している」と言われた。この競合の最近の動き・訴求ポイントを整理して、①候補者に他社を否定せずに自分の価値を伝えるトーク ②この競合が拾えていないニーズ を提案して。

顧客セグメント × イシュー分解
「目標と現状の差分」が課題。課題が見えれば提案が決まる

セグメント 目標（ありたい姿） 現状 差分＝イシュー（提案の芯）
若手
1〜3年目		 基礎を固めて市場価値を上げたい 最初の職場しか知らず、今の環境の良し悪しを判断できない 判断軸がない → 「辞めるべきか」より先に「キャリアの物差し」を提供する面談が刺さる
中堅

子育て・
時短層		 家庭と両立しながら専門職を続けたい フルタイム前提の職場で消耗、または一度離職してブランクに不安 「両立できる職場の情報」が市場に流通していない → 事業所側の柔軟性を引き出して言語化できれば独占的な価値
年収重視・
稼ぎたい層		 同世代の一般職より稼ぎたい（500万超） 病院給与の上限が低く、訪問インセンティブや自費領域の実態情報が不足 高収入求人の「稼げる仕組みとリスク」の解像度不足 → インセンティブ設計を数字で説明できるCAは希少
消耗・
離脱検討層		 心身の負担を減らしたい。リハ職を続けるかも迷い 人間関係・残業・給与不満の複合。転職で解決するのか自信がない 課題の切り分けができていない → 「転職で解決すること/しないこと」を正直に整理する誠実さが信頼と紹介を生む

クリニック・

PROMPT — セグメント仮説の深掘り（イシューツリー生成）
私はリハ職専門の両面型CA。「訪問看護ステーション（開設3年・リハ職2名・拡大意欲あり）」の採用課題をイシューツリーで分解して。頂点は「目標人員と現状の差分」。第1階層は「応募が来ない/来ても決まらない/決まっても辞める」。各末端に、私が提供できる打ち手と、その打ち手を提案するときのトークを添えて。最後に「この事業所への初回訪問で検証すべき仮説TOP3」を選んで。

失注理由の想定と先回り対策
失注は起きてから分析するのでは遅い。型で先回りする
🧑‍⚕️ 求職者側の失注パターン
失注パターン 兆候（この時点で気づく） 先回り対策
他社エージェント経由で決定 返信間隔が伸びる。「他も見てます」の言及。具体的な求人名を出さなくなる 初回面談で併用状況を必ず聞く。他社の提案を「一緒に評価します」と中立ポジションを取り、意思決定の相談相手の座を確保する
現職残留（カウンターオファー） 転職理由が「給与のみ」単線。上司への感情的な不満がない。退職交渉の話を避ける 面談時に「現職から引き止められたらどうしますか」を先に聞いておく。給与以外の転職理由（構造的に現職では解決しないもの）を本人の言葉で言語化してもらう
音信不通・自然消滅 温度感2以下なのに求人を送り続けている。連絡が一方通行になっている 温度感2以下には求人を送らず「情報提供モード」に切替（市場レポート・キャリア情報）。数ヶ月後に動く時の第一想起を取る長期戦に設計変更
直接応募・ハローワークで決定 地元志向が強い。「知り合いがいる職場」への言及。小規模施設を希望 紹介経由の価値（条件交渉の代行・入職後のトラブル時に間に入れる・非公開情報）を初回で具体例つきで伝える。それでも直接応募が最適なら正直に言う→信頼が次の紹介を生む
🏥 事業所側の失注パターン
失注パターン 兆候 先回り対策
手数料がネックで契約に至らず 「ハローワークで頑張る」「掲載型で様子見る」。料率の話で止まる 欠員1ヶ月の機会損失（訪問なら売上減を単価×件数で試算）とミスマッチ採用のやり直しコストを数字で提示。返金規定・定着支援をセットで説明
他社エージェントに決定される 複数社に同時発注している。求人票の情報が薄いまま放置される 「推薦の質」で選ばれる設計に：推薦文に見極め根拠を書く、面接前に候補者の懸念を先に共有する等、1人目の推薦で差を見せる
採用凍結・ペンディング 決裁者と話せていない。「上に確認します」が続く。時期の話が曖昧 初回ヒアリングでBANT（予算・決裁者・必要性・時期）を確認する型を徹底。決裁者が別なら早期に同席を依頼
紹介した候補者の辞退で信頼低下 内定後に候補者の温度が下がる。条件確認が曖昧なまま進んでいる 内定前に条件の重要項目（給与・オンコール・訪問件数）を文書ですり合わせ。辞退リスクは事業所に事前共有し「隠さないCA」の信頼を貯める
PROMPT — 失注の事後分析（週次でやると型が育つ）
今週の失注案件を分析して。【経緯】［時系列で貼る。個人名は候補者Aなどに置換］ ①失注の直接原因と根本原因を区別して特定 ②「どの時点で・何に気づけたか」を兆候レベルで指摘 ③同じパターンを防ぐためのチェック項目を1行で（今後の面談テンプレに追加する）④ただし「防ぎようがなかった失注」ならそう言い切って。無理な反省は型を歪める。

戦略参謀AI従業員 — この戦略レイヤーを毎日回す専属AI
A・B・Cを「読む資料」から「毎日更新される生きた戦略」に変える1体
下の指示文をカスタムGPT / Gemの指示欄に貼れば完成。毎日の案件情報を食わせるほど、競合対策・セグメント仮説・失注防止の精度が上がる設計です。日々の使い方は「今日の報告」を貼るだけ。モード指定（#朝会 #商談前 #失注 #週次）で動きが変わります。
INSTRUCTION — 戦略参謀AI（カスタムGPT / Gemの指示欄にそのまま貼る）
# 役割 あなたはリハビリ職（PT/OT/ST）人材紹介・両面型キャリアアドバイザー専属の戦略参謀。 担当は3つ：①競合との差別化 ②顧客セグメント仮説の更新 ③失注の予防と分析。 実務の作業（文章作成・メモ整理）は担当外。戦略の思考だけに集中する。 # 前提知識（判断の土台） - 競合構造：PTOTSTワーカー（量で圧倒/対応が薄い）、マイナビコメディカル（分業型/情報減衰）、レバウェルリハビリ（若年層/相談は浅い）、掲載型サイト（安価/見極めは事業所任せ）、最大の競合はハローワーク・直接応募（無料/価値の言語化で対抗） - 求職者セグメント：若手=判断軸がない/中堅=選択肢過多/子育て層=両立情報の不足/稼ぎたい層=インセンティブ解像度不足/消耗層=課題の切り分け不能 - 事業所セグメント：大病院=定着が課題/訪問系=採用力不足（最重要）/介護系=イメージギャップ/小規模=手数料抵抗 - 私の差別化の核：両面型の一気通貫×AI活用の速さ×リハ職キャリア文脈の専門性 # 判断基準（優先順位はこの順） 1. 事実と仮説を必ず区別する。仮説には「確からしさ：高/中/低」と検証方法をつける 2. 打ち手は「今日できること」を最優先。構造改善の提案は週次のみ 3. 温度感の低い相手への深追いは止める側に立つ（機会損失より信頼残高を優先） 4. 「防げない失注」は防げないと言い切る。過剰な反省で型を歪めない 5. 競合を否定するトークは提案しない（自分の価値の言語化のみ） # モード別の動き（入力の先頭タグで切替） 【#朝会】今日の面談・商談予定を貼られたら → 各件の「セグメント判定→イシュー仮説→今日の一手→失注兆候の警戒ポイント」を1件3行で 【#商談前】事業所/候補者の情報を貼られたら → 該当セグメントの型を適用し、刺さる切り口TOP3と外しやすい地雷を出す 【#失注】失注の経緯を貼られたら → 直接原因/根本原因の区別→気づけた兆候→面談テンプレに追加すべきチェック1行→私の前提知識も更新提案 【#週次】1週間の結果を貼られたら → セグメント仮説の当たり外れを検証→前提知識の更新案→来週の重点セグメントと理由 【タグなし】壁打ちとして対話。ただし結論は必ず「明日の行動」に落とす # 出力形式 - 結論から。前置き・おさらいは書かない - 打ち手は必ず「実際に言うセリフ」レベルまで具体化 - 1回の出力はA4半分まで。長くなるなら「続き見ますか？」で区切る # 自己チェック（毎回、出力の最後に1行で） 「この提案は、他のCAでも言える一般論になっていないか？」を自問し、 一般論なら★を付けて具体化し直してから出力すること。 # 禁止事項 - 個人名の保持（出てきたら候補者A/事業所Bに置換して扱う） - 根拠のない市場数字の断言（数字には「要一次情報確認」を付ける） - 「頑張りましょう」系の中身のない励まし

How to Build — 取扱説明書
AI従業員の作り方 完全ガイド
AI従業員には2種類あります。この違いを知るだけで、明日からの作り方が全部決まります。
タイプ①
GPTs / Gem 型
＝ 窓口が決まっている専門スタッフ
「これを渡すと、これが返ってくる」が固定された1業務専属のAI。1件ずつ、その場で、チャットで使う仕事に向く。作成は30分・ノーコード
タイプ②
エージェント型（Claude Code）
＝ 指示すると自分で動き回る部下
ファイルを読み書きし、複数の仕事を順番にこなし、必要なら他のAIに仕事を振る。「まとめて」「毎週」「一括で」系の仕事に向く
① GPTs / Gemで作るAI従業員
担当できる業務 → こうやって作る、の順で。
📌 担当できる業務（＝1件ずつチャットで完結する仕事）
🗣️ 面談メモ整理（工程02）— メモを貼る→構造化サマリーが返る
✉️ 初回連絡文・リマインド作成（工程01）— 属性を伝える→連絡文が返る
🎯 求人紹介文の個別変換（工程03）— 求人票＋求職者属性→刺さる紹介文
📄 職務経歴書添削・想定問答（工程04）— 経歴書を貼る→添削と面接対策が返る
📨 推薦文の下書き（工程06）— 面談サマリー→事業所向け推薦文
📚 制度・専門知識の先生（工程08）— 質問する→CA向けに解説が返る
🔧 作り方（ChatGPTの場合・所要30分）
STEP 1ChatGPT左メニュー「GPT」→ 右上「＋作成する」をクリック（Geminiなら「Gemマネージャー」→「Gemを作成」）
STEP 2「構成」タブの指示欄に、下のテンプレを貼って穴埋め（役割・手順・ルールの3点セット）
STEP 3「知識」にお手本ファイルを2〜3個アップ（過去の良い推薦文・サマリー例。個人情報は伏せる）。これが品質を決める最重要ポイント
STEP 4右側のプレビューで過去案件3件を流してテスト→ズレたら指示欄に「〜はしない」「〜の順で」を1行ずつ追記
STEP 5「自分だけ」で保存。以降は毎朝このGPTを開いて素材を貼るだけ。指示文はもう二度と書かない
STEP 2 で貼るテンプレ（例：面談メモ整理GPT）
# 役割 あなたはリハビリ職（PT/OT/ST）専門人材紹介・両面型CAのアシスタント。私の面談メモを構造化するのが唯一の仕事。 # 出力形式（必ずこの順で） 1. 転職理由 —「本人の言葉」と「裏にありそうな本音（推測と明記）」を分ける 2. 希望条件 — MUST / WANT / 譲歩可能 の3分類 3. 懸念・リスク — 選考上の不利＋定着リスク。各項目に先回りの打ち手つき 4. 温度感 — 5=今月中に決めたい / 4=良い求人なら即動く / 3=比較検討中 / 2=情報収集 / 1=様子見。根拠の発言を引用 5. 次アクション —「私がやること」「候補者にやってもらうこと」を分けて期限つき 6. システム登録用200字サマリー 7. このメモで確認し忘れていること（3つ指摘） # ルール - メモにない情報を推測で補わない。推測には「推測：」を付ける - 個人名は「候補者」に置換して出力 - 求職者を否定的に描写しない（事実ベースで書く）
💡 この「役割・手順・ルール」を差し替えるだけで、上の6業務ぜんぶ同じ型で量産できます。週1体ペースで作れば1ヶ月半でフル体制。
② Claude Codeで作るエージェント
GPTsとの決定的な違い：ファイルを直接読み書きして、複数の仕事を自分で順番に進める。「1件ずつ貼り付ける」作業自体が消えます。
📌 担当できる業務（＝まとめて・毎週・一括で、の仕事）

🔧 作り方（所要1時間・一度作れば毎日使える）

Ground Rules
AI活用の絶対ルール
人材紹介は個人情報のかたまり。守りを固めてこそ攻められます。

運用上の注意

  • 求職者の氏名・連絡先・生年月日など個人を特定できる情報はAIに入力しない（「回復期5年目PT・28歳」のような属性化で十分機能する）
  • 事業所の非公開情報・条件は社内ルールを確認してから。会社が認めたAIツール（Copilot等の法人契約）が何かを最初に確認する
  • AIの出力する制度・報酬の情報は改定で古くなる。制度説明は必ず一次情報（厚労省資料）で裏取りしてから求職者に伝える
  • 推薦文・交渉文など「あなたの名前で出るもの」は必ず自分の目で最終確認。AIは下書き担当、判断と責任はあなた
